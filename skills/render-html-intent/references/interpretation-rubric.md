# Interpretation Rubric

Use this when a natural-language request does not clearly specify the final HTML shape.

## Choose The Artifact

- "Track", "manage", "organize", "CRM", "admin", "workflow": build an app-like operational surface with tables, filters, status, and actions.
- "Explain", "teach", "summarize", "report", "brief": build a readable document page with hierarchy, evidence, callouts, and navigation aids.
- "Compare", "choose", "rank": build comparison cards or a matrix with clear criteria and a recommendation area.
- "Visualize", "map", "flow", "process": build a diagram-like page with SVG, CSS layout, or structured stages.
- "Landing", "launch", "sell", "promote": build a first-viewport page where the product or offer is immediately visible.
- "Calculator", "simulator", "quiz", "generator": build an interactive tool with inputs, results, empty states, and validation.

## Fill Missing Details

- Invent sample data that makes the page useful to inspect.
- Use neutral but domain-specific labels instead of generic "Item 1" copy.
- Prefer three to six meaningful sections over many shallow sections.
- Use a restrained palette with one accent and clear neutrals; avoid one-hue pages unless the brand requires it.
- Include realistic responsive behavior: stacked mobile layout, readable line lengths, and controls that do not shrink below usable size.

## Final Review Questions

- Does the first viewport show the main artifact, not instructions about the artifact?
- Can someone infer what the page does without reading the original prompt?
- Are the most important actions or insights visually dominant?
- Would the page still be coherent with different viewport widths?
- Are there unresolved placeholders, broken assets, or decorative elements that do not serve the intent?
