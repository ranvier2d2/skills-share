# Linear Publication Flow

## Goal

Leave durable, reviewable evidence on the Linear issue.

## Preferred sequence

1. Record the demo.
2. Confirm the artifact path and file size.
3. Attach the artifact to the issue.
4. Add a short comment that explains the evidence.

## Comment standard

The comment should state:
- what was validated
- what path was shown
- any known limitation

Avoid vague comments like:
- demo attached
- works now

Prefer comments like:
- Demo video attached.
- Validated: settings page saves a renamed workspace successfully.
- Path shown: open settings, rename workspace, save, reload, confirm persisted name.
- Known limitation: mobile layout was not covered in this run.

## Operator expectation

A reviewer should be able to open the issue and understand the evidence without replaying the whole implementation history.
